# 📋 Инструкция по подключению ЮKassa (для самозанятых)

## ⚡ Быстрый старт (2-3 дня)

### Шаг 1: Регистрация в ЮKassa

1. Перейди на сайт **[yookassa.ru](https://yookassa.ru)**
2. Нажми **"Подключиться"** → **"Физическому лицу / Самозанятому"**
3. Заполни заявку:
   - **ФИО**: Князькова Екатерина Сергеевна
   - **Email**
   - **Телефон**
   - **ИНН** (обязательно)
4. Загрузи документы:
   - 📄 Паспорт (разворот + прописка)
   - 📄 Справка о статусе самозанятого (из приложения "Мой налог")
   - 📄 ИНН
5. Ожидай модерации (**2-3 рабочих дня**)

---

### Шаг 2: Получение API-ключей

После одобрения заявки:

1. Войди в личный кабинет ЮKassa
2. Перейди в **Настройки → API-ключи**
3. Скопируй:
   - **shopId** (идентификатор магазина)
   - **secretKey** (секретный ключ)

⚠️ **Важно:** secretKey показывается только один раз! Сохрани его надёжно.

---

### Шаг 3: Настройка backend-сервера

#### 3.1 Установка зависимостей

```bash
cd backend
npm install
```

#### 3.2 Создание .env файла

1. Скопируй `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```

2. Открой `.env` и заполни:
   ```env
   # Порт сервера
   PORT=3000

   # URL сайта (для return_url)
   SITE_URL=https://psixolog-site.ru

   # ЮKassa (из личного кабинета)
   YOOKASSA_SHOP_ID=твой_shopId
   YOOKASSA_SECRET_KEY=твой_secretKey

   # Telegram для уведомлений
   TELEGRAM_BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
   TELEGRAM_CHAT_ID=-1001234567890

   # Режим (test/production)
   PAYMENT_MODE=test
   ```

---

### Шаг 4: Настройка Telegram-бота

#### 4.1 Создание бота

1. В Telegram найди **[@BotFather](https://t.me/BotFather)**
2. Отправь команду `/newbot`
3. Введи имя бота (например: `Psixolog Payment Bot`)
4. Введи username бота (например: `EkaterinaPsixologBot`)
5. Скопируй полученный **токен** (вида `123456789:ABCdef...`)

#### 4.2 Получение Chat ID

**Вариант A: Личный чат**

1. Найди бота **[@userinfobot](https://t.me/userinfobot)**
2. Нажми "Start"
3. Скопируй полученный ID (например: `123456789`)

**Вариант B: Группа/канал**

1. Добавь бота в группу/канал
2. Дай права администратора
3. Отправь любое сообщение в группу
4. Перешли сообщение боту **[@chatid_echo_bot](https://t.me/chatid_echo_bot)**
5. Скопируй ID (начинается с `-100`)

---

### Шаг 5: Запуск сервера

#### Локально (для тестирования):

```bash
cd backend
npm run dev
```

Сервер запустится на `http://localhost:3000`

#### На хостинге:

**Вариант A: Vercel (бесплатно)**

1. Создай файл `vercel.json`:
   ```json
   {
     "version": 2,
     "builds": [{ "src": "server.js", "use": "@vercel/node" }],
     "routes": [{ "src": "/(.*)", "dest": "server.js" }]
   }
   ```

2. Установи CLI:
   ```bash
   npm i -g vercel
   ```

3. Задеплой:
   ```bash
   vercel login
   vercel
   ```

4. Добавь переменные окружения в панели Vercel:
   - `YOOKASSA_SHOP_ID`
   - `YOOKASSA_SECRET_KEY`
   - `TELEGRAM_BOT_TOKEN`
   - `TELEGRAM_CHAT_ID`
   - `SITE_URL`

**Вариант B: Railway (бесплатно)**

1. Зарегистрируйся на **[railway.app](https://railway.app)**
2. Создай новый проект
3. Подключи GitHub репозиторий
4. Добавь переменные окружения в панели
5. Railway автоматически задеплоит

**Вариант C: Свой VPS**

1. Арендуй VPS (Timeweb, Selectel, Reg.ru)
2. Установи Node.js:
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
   sudo apt-get install -y nodejs
   ```

3. Загрузи файлы сервера
4. Установи зависимости:
   ```bash
   npm install
   ```

5. Запусти через PM2:
   ```bash
   npm install -g pm2
   pm2 start server.js --name psixolog-payment
   pm2 save
   pm2 startup
   ```

---

### Шаг 6: Настройка webhook в ЮKassa

1. В ЛК ЮKassa перейди в **Настройки → HTTP-уведомления**
2. Нажми **"Добавить webhook"**
3. Укажи URL:
   ```
   https://your-backend-url.vercel.app/api/webhook
   ```
   или
   ```
   https://api.psixolog-site.ru/api/webhook
   ```

4. Выбери события:
   - ✅ `payment.succeeded` — оплата успешна
   - ✅ `payment.canceled` — оплата отменена
   - ✅ `payment.waiting_for_capture` — ожидает подтверждения

5. Сохрани

---

### Шаг 7: Настройка frontend

#### 7.1 Обновление URL backend

Открой `js/main.js` и замени:

```javascript
const BACKEND_URL = 'https://your-backend-url.ru';
```

на свой URL:

```javascript
const BACKEND_URL = 'https://psixolog-payment.vercel.app';
// или
const BACKEND_URL = 'https://api.psixolog-site.ru';
```

#### 7.2 Обновление payment-success.html

Открой `payment-success.html` и замени:

```javascript
const BACKEND_URL = 'https://your-backend-url.ru';
```

на тот же URL.

### Шаг 8: Тестирование

#### Тестовый режим

1. В ЛК ЮKassa включи **тестовый режим**
2. Используй тестовые карты:
   - **Успешная оплата**: `1111 1111 1111 1026`
   - **Отказ**: `1111 1111 1111 1010`
   - Любые срок действия и CVV

#### Проверка работы

1. Открой сайт
2. Нажми **"Оплатить картой — 3500 ₽"**
3. Должна открыться платёжная страница ЮKassa
4. Введи тестовые данные карты
5. После оплаты:
   - ✅ Должна открыться страница успеха
   - ✅ Тебе должно прийти уведомление в Telegram

## 🔄 Рабочий процесс (полная схема)

```
1. Клиент на сайте → жмёт "Оплатить картой"
         ↓
2. Frontend → POST /api/create-payment (твой сервер)
         ↓
3. Сервер → API ЮKassa (создаёт платёж)
         ↓
4. ЮKassa → возвращает ссылку на оплату
         ↓
5. Сервер → отдаёт ссылку frontend
         ↓
6. Клиент → переходит → платит на стороне ЮKassa
         ↓
7. ЮKassa → обрабатывает платёж
         ↓
8. ЮKassa → POST /api/webhook (твой сервер)
   │
   ├─→ Формирует чек (автоматически для самозанятого)
   ├─→ Чек уходит клиенту на Email/SMS
   └─→ Чек уходит в налоговую (автоматически)
         ↓
9. Сервер → уведомление в Telegram психологу
         ↓
10. Психолог → подтверждает запись клиенту
```

## ❓ Частые вопросы

### Q: Сколько времени занимает подключение?
**A:** Регистрация + модерация 2-3 дня. Настройка сервера — 1-2 часа.

### Q: Нужна ли онлайн-касса?
**A:** ❌ Нет! ЮKassa сама формирует чеки для самозанятых.

### Q: Как вернуть деньги клиенту?
**A:** В ЛК ЮKassa найди платёж → "Вернуть" → укажи сумму.

### Q: Можно ли принимать оплату рекуррентно (подписка)?
**A:** Да, но требуется отдельная настройка (сохранение payment_method).

### Q: Что если клиент оплатил, но не записался?
**A:** Уведомление придёт в Telegram. Свяжись с клиентом самостоятельно.
